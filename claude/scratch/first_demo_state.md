# first_demo branch — State

## Goal
Build demos showcasing llmos-v0 primitives: a book-search agent and a chat-with-history agent.

## Current Status
Chat-with-history demo (M0-M5) complete. 20 tests passing. CLI works in mock mode with history search. Ready for user review.

## Demos

### Book Search (`examples/book-search/`)
Epub → chunks → map-reduce search → web UI visualization. All milestones complete. UI uses Tailwind v4 + shadcn/ui with armandmcqueen.dev light theme.

### Chat with History (`examples/chat-history/`) — NEW
Interactive CLI chat where AI can search previous conversations. Demonstrates tool-use loops, task spawning from tools, and store as persistent memory.

**Milestones:**
- **M0** ✅ Project scaffolding + types
- **M1** ✅ History discovery + tests (8 tests)
- **M2** ✅ Search map-reduce + tests (5 tests)
- **M3** ✅ Chat turn with tool loop + tests (7 tests)
- **M4** ✅ Interactive CLI
- **M5** ✅ README + DESIGN docs

**Key files:**
- `src/types.ts` — ChatTurnEvent, SearchEvent, etc.
- `src/history.ts` — readAllChatTurns() — discovers chat turns from task metadata
- `src/search.ts` — searchCoordinatorTask + searchWorkerTask (map-reduce)
- `src/chat.ts` — chatTurnTask with manual tool-use loop
- `src/search-tool.ts` — createSearchHistoryTool(ctx) factory (closure over TaskContext)
- `src/mock-ai.ts` — MockAI with tool-call simulation
- `src/setup.ts` — createChatLLMOS(mode, apiKey?, delayMs?)
- `src/cli.ts` — Interactive readline CLI with /history, /inspect, /save, /quit
- `store-snapshot.json` — Generated by CLI runs

**Key design decisions:**
- Manual tool-use loop (not maxSteps) — no core changes, better per-call logging
- History discovery via kernel task metadata scan — no separate index
- Tool constructed as closure over TaskContext for ctx.spawn access
- All Haiku models
- CLI first, web UI deferred
- MockAI keyword matching + tool-call simulation
